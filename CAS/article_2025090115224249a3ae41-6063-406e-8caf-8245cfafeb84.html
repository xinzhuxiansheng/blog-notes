单点登录Apereo CAS 7.1客户端登出配置及免认证页面问题

从上一篇博客中，我们已经实现了CAS客户端集成和服务端授权，能够通过统一认证登录访问系统资源了，接下来我们讲一下如何实现CAS客户端登出及免认证页面配置的问题，还有以个人见解讲一下CAS和Spring security的关系。

上一篇博客：单点登录Apereo CAS 7.1客户端集成教程-CSDN博客

客户端登出

添加logout接口

我们添加一个logout接口，这里主要是两个步骤：1、销毁当前的会话；2、跳转到cas的logout，通知CAS登出。

其中这里获取会话使用的是 request.getSession(false)，如果直接使用request.getSession()，那么在没有会话的时候，会自动创建一个会话，这就没必要了。

    @RequestMapping("/logout")
    public void logout(HttpServletRequest request, HttpServletResponse response) throws IOException {
        // 获取会话
        HttpSession session = request.getSession(false);
        if (session != null) {
            // 销毁会话
            session.invalidate();
        }
        // 重定向到登出成功页面
        response.sendRedirect("http://localhost:8443/cas/logout");
    }

现在我们登录后，再访问http://localhost:8090/logout ，系统会注销会话并，然后跳转到cas的logout页面，页面提示已经注销成功。重新访问应用，已经提示需要重新登录。

但是这里有个问题，我们如果想在登出后重新登录应用，在这个页面点击登录，那么访问的页面是cas的统一认证，并不是我们的应用。

登出后自动回到应用

logout添加service

为了实现退出登录后重新回到应用，我们可以在跳转cas的logout后面添加登出后跳转的地址，这里的地址需要进行‌URL编码（URL encoding）。

?service=http%3A%2F%2Flocalhost%3A8090%2F

修改application.properties

添加了上述代码后，退出登录依然会停留在cas的logou页面，我们可以修改cas服务端的application.properties文件，并加入下面这行代码，就可以实现登出后，直接跳转到应用页面。

cas.logout.followServiceRedirects=true

现在我们登出后，系统会直接回到应用的登录界面了。实际上，这是进行了两步跳转，logout→应用页，发现未登录→跳转到登录页。

设置免登录页面（不推荐）

我们退出登录后，大部分系统会有自己的logout页面提示，像上面的cas的logout页面一样，会提示用户已经成功退出登录了，如果需要再点击系统页面重新登录。所以我们需要设置有页面是免授权就能访问的。这里之所以写不推荐，是因为这里有安全漏洞，我们还是需要了解一下，以免以后踩坑，具体在后面讨论。

添加logoutSuccess页面

现在我们添加一个logoutSuccess接口，为了方便退出登录后重新访问，我们还加了一个Home的超链接，方便回到主页。

    @ResponseBody
    @RequestMapping("/logoutSuccess")
    public String logoutSuccess(Model model) {
        return "Logout Success" +
                "<br>" +
                "<a href='/'>Home</a>" +
                "<br>";
    }

修改logout跳转页面

现在我们修改前面的logout接口，将最后的service，修改到logoutSuccess页面。

response.sendRedirect("http://localhost:8443/cas/logout?service=http%3A%2F%2Flocalhost%3A8090%2FlogoutSuccess");

添加一个cas的配置类

现在我们在项目中添加一个config包，后续放我们的一些配置文件。现在新建一个实现了CasClientConfigurer接口的CasClientConfigurerImpl类。将/logoutSuccess接口设置未忽略的，这样访问这个接口，就不需要登录了。

package cn.wuyitian.config;

import java.util.Map;
import org.apereo.cas.client.boot.configuration.CasClientConfigurer;
import org.springframework.boot.web.servlet.FilterRegistrationBean;
import org.springframework.stereotype.Component;

/**
 * cas-client-support-springboot 依赖提供了CAS客户端的自动配置，
 * 当自动配置不满足需要时，可通过实现{@link CasClientConfigurer}接口来重写需要自定义的逻辑
 */
@Component
public class CasClientConfigurerImpl implements CasClientConfigurer {
    /**
     * 配置认证过滤器，添加忽略参数，使/logoutSuccess登出提示页免登录
     */
    @Override
    @SuppressWarnings({ "rawtypes", "unchecked" })
    public void configureAuthenticationFilter(final FilterRegistrationBean authenticationFilter) {
        Map initParameters = authenticationFilter.getInitParameters();
        initParameters.put("ignorePattern", "/logoutSuccess");
    }
}

现在我们重新访问logout接口，发现已经成功跳转到了logoutSuccess页面，我们点击Home，才会回到需要登录的主页。

免登录页面的安全问题

上面的方式之所以不安全，主要是我们通过ignorePattern添加的接口，是一种字符串匹配机制。具体说，就是你的url只要包含了“/logoutSuccess”这个字符串，都会被当成免授权页面。也就是说，只要知道了这个字符串，就可以访问系统内的任意页面。

攻击示例

在我们的系统的设计上，/user这个接口，应当是登录后才能访问的。现在我们在url中添加“?/logoutSuccess”，发现不用登录就可以直接访问了。

CAS与Spring Security

一开始，笔者以为使用了CAS已经完成了登录授权，客户端是可以拿到用户名的，CAS本身有它的安全机制，Spring Security也是做授权认证的。如果没有特殊要求，客户端就不用Spring Security这些，可以直接在客户端上做业务的工作了。

事实上，CAS与Spring Security是相互独立的。CAS只负责单点登录认证，而客户端具体的安全认证，比如用户角色、方法级鉴权，这些都还是需要Spring Security来完成。

这样说可能有点抽象，我们将CAS比喻成警察局，客户端比喻成企业、学校、小区，Spring Security是物业行业里的一间公司。警察局可以开具居民身份证或其他的身份合法证明，但是一个人在企业什么职位，能进入哪个小区哪个单元，都是由企业和小区各自的物业管理的。CAS的意义，就是小区物业可以直接通过身份证比对个人信息，自行查验后放行，不用业主再证明自己是谁。

我们前面的免认证页面，也可以理解为小区的开放花园，应当由小区物业自行管理设置围栏，而不是随便一个人说我要去花园，就可以从其他业主的客厅穿过去。
